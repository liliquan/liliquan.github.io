CSRF

CSRF（Cross-Site Request Forgery）跨站请求伪造，通过伪造用户的请求来攻击网站、盗取用户的信息。

原理

CSRF 的攻击原理比较巧妙，它的流程是：

  1. 用户在 A 网站中登录了
  2. 这时用户访问 B 网站
  3. B 网站中偷偷访问了 A 网站中一个必须登录才能访问的网址（比如：转账 、删除自己的数据等）

示例：比如有了个 B 站，中有以下代码，那么当你访问这个页面时，它会就在后台偷发一个请求到 A 站：

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Document</title>
</head>
<body>

    <h1>欢迎 b 站</h1>

    <iframe style="display:none;" src="" name="abc"></iframe>

    <form target="abc" id="a" style="display:none;" method="post" action="http://localhost:9999/blog/store">
        <input type="text" name="title" value="你被攻击 ~">
        <textarea name="content" >你被攻击了，被攻击了，攻击了，击了，了，</textarea>
        <input type="text" name="is_show" value="1">
    </form>

</body>
</html>

<script>
document.getElementById("a").submit();
</script>

防御

防御的原理：判断一个请求是否是由本站发生的（防止外站提交）。

实现思路：

  • 每个用户在访问网站时，程序上生成一个随机的字符串，我们管它叫令牌（token）
  • 然后把这个令牌放到本站的表单中
  • 服务器在处理表单时先判断有没有令牌
  • 因为黑客无法拿到别人的令牌，也就无法伪造表单

实现流程：

1、程序上生成一个随机的字符串（令牌），保存到 SESSION 中

说明：每个用户在访问网站时，服务器上都会为这个用户创建一个 SESSION 数组，每个用户有一个自己的 SESSION，所以我们应该把每个用户的令牌保存到它的 SESSION 。

l

public/index.php

function csrf()
{
    if(!isset($_SESSION['token']))
    {
        // 生成一个随机的字符串
        $token = md5( rand(1,99999) . microtime() );
        $_SESSION['token'] = $token;
    }
    return $token;
}

2、在所有的表单中都使用这个函数生成 令牌并放到表单中

网页中所有的以 POST 方式提交的表单，都要添加令牌：

<input type="hidden" name="_token" value="<?=csrf()?>">

3、网站中添加验证令牌的代码

因为在最开始就要验证所以必须要控制器中的代码执行之前验证。

public/index.php

在最前面添加以下代码：

<?php
// 使用 redis 保存 SESSION
ini_set('session.save_handler', 'redis');
// 设置 redis 服务器的地址、端
ini_set('session.save_path', 'tcp://127.0.0.1:32768?database=3');

session_start();

// 如果用户以 POST 方式访问网站时，需要验证令牌
if($_SERVER['REQUEST_METHOD'] == 'POST')
{
    if(!isset($_POST['_token']))
        die('违法操作！');

    if($_POST['_token'] != $_SESSION['token'])
        die('违法操作！');
}

总结：

  1. 所有重要的操作要以 POST 方式实现（只对 POST 验证了令牌）（比如：添加、删除、修改、加好友、转账、点赞等等）
  2. POST的表单中要调用 csrf 函数添加令牌
